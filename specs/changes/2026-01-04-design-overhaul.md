# 変更記録: 2026-01-04 デザイン・レイアウト刷新

## 1. 変更の理由 (Reason)
- 現状のデザインが簡素であり、`shadcn/ui` のようなモダンな外観への改善要望があったため。
- 設定画面がサイドバーと競合し、表示領域が狭くなってしまう（3カラム問題）レイアウト上の課題を解決するため。
- ダークモード/ライトモードにおけるプルダウンメニュー等の視認性を向上させるため。
- 今後のスタイル保守性を高めるため、CSS変数ベースのデザインシステムを確立するため。

## 2. 変更内容 (Changes)

### 2.1 デザインシステムの実装
- `index.css` に HSL ベースの CSS 変数（Zinc テーマ）を導入。
- Tailwind CSS の標準カラーを可能な限りセマンティック変数に置き換え。

### 2.2 レイアウトの変更
- `App.tsx` において、設定画面、コマンド管理、ファイル管理を全画面オーバーレイ（絶対配置）に変更。
- これにより、サブ画面がサイドバーの上に重なる形で表示され、広い表示領域を確保。

### 2.3 コンポーネントの調整
- `Sidebar.tsx`, `SettingsView.tsx`, `ModelSettings.tsx` 等のコンポーネントにおけるハードコードされた色指定を CSS 変数に置き換え。
- プルダウンメニュー (`select`, `option`) の配色をテーマに追従するように修正。

## 3. 影響範囲
- 全てのナビゲーションフロー（テストによる検証を必須とする）。
- ライト/ダークモードの表示。
